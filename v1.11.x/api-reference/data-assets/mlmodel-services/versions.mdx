# ML Model Service Versions

Every change to an ML model service entity creates a new version. Use these endpoints to view the version history and retrieve specific versions.

## List Versions

<ParamField path="id" type="string" required>
  UUID of the ML model service.
</ParamField>

## Get Specific Version

Use `GET /v1/services/mlmodelServices/{id}/versions/{version}` to retrieve a specific version.

<ParamField path="id" type="string" required>
  UUID of the ML model service.
</ParamField>

<ParamField path="version" type="string" required>
  Version number to retrieve (e.g., `0.2`).
</ParamField>

<RequestExample dropdown>
```python GET /v1/services/mlmodelServices/{id}/versions
from metadata.sdk import configure
from metadata.sdk.entities import MlModelServices

configure(
    host="https://your-company.open-metadata.org/api",
    jwt_token="your-jwt-token"
)

# List all versions
versions = MlModelServices.get_versions("ca22d46e-81b9-4e48-85b5-0adc44980da9")
for v in versions:
    print(f"Version {v.version}: {v.description}")

# Get a specific version
svc_v1 = MlModelServices.get_specific_version(
    "ca22d46e-81b9-4e48-85b5-0adc44980da9",
    "0.1"
)
print(f"Original description: {svc_v1.description}")
```

```java GET /v1/services/mlmodelServices/{id}/versions
import static org.openmetadata.sdk.fluent.MlModelServices.*;

// List all versions
var history = MlModelServices.getVersions("ca22d46e-81b9-4e48-85b5-0adc44980da9");

for (var version : history.getVersions()) {
    System.out.println("Version: " + version);
}

// Get a specific version
var v1 = MlModelServices.getSpecificVersion(
    "ca22d46e-81b9-4e48-85b5-0adc44980da9",
    0.1
);
System.out.println("Description: " + v1.getDescription());
```

```bash GET /v1/services/mlmodelServices/{id}/versions
# List all versions
curl "{base_url}/api/v1/services/mlmodelServices/ca22d46e-81b9-4e48-85b5-0adc44980da9/versions" \
  -H "Authorization: Bearer {access_token}"

# Get a specific version
curl "{base_url}/api/v1/services/mlmodelServices/ca22d46e-81b9-4e48-85b5-0adc44980da9/versions/0.1" \
  -H "Authorization: Bearer {access_token}"
```

## Returns

**List versions** returns an object with `entityType` and a `versions` array of serialized entity snapshots (newest first).

**Get specific version** returns the full ML model service object as it existed at that version.

## Error Handling

| Code | Error Type | Description |
|------|-----------|-------------|
| `401` | `UNAUTHORIZED` | Invalid or missing authentication token |
| `404` | `NOT_FOUND` | ML model service or version does not exist |

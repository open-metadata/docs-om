# Discovery

The Discovery API provides full-text search and field-level querying across all entity types in your metadata catalog. Under the hood, OpenMetadata uses **Elasticsearch** to index and search metadata, giving you powerful query capabilities including fuzzy matching, filters, and aggregations.

## Available Endpoints

<CardGroup cols={2}>

<Card title="Search Entities" icon="magnifying-glass" href="/v1.11.x/api-reference/discovery/search">
  Full-text search across all entity types using Elasticsearch query syntax. Supports pagination, sorting, filtering, and index-specific searches.
</Card>

<Card title="Field Query" icon="filter" href="/v1.11.x/api-reference/discovery/field-query">
  Search for entities by specific field values such as column names, tags, owners, or service names.
</Card>

</CardGroup>

## Search Indexes

OpenMetadata maintains separate search indexes for each entity type. You can search across all indexes or target a specific one:

| Index | Description |
|-------|-------------|
| `all` | Search across all entity types |
| `table_search_index` | Tables |
| `topic_search_index` | Topics (Kafka, etc.) |
| `dashboard_search_index` | Dashboards |
| `pipeline_search_index` | Pipelines |
| `mlmodel_search_index` | ML Models |
| `container_search_index` | Storage containers |
| `search_entity_search_index` | Search indexes |
| `glossary_term_search_index` | Glossary terms |
| `tag_search_index` | Tags and classifications |
| `user_search_index` | Users |
| `team_search_index` | Teams |
| `domain_search_index` | Domains |
| `data_product_search_index` | Data products |

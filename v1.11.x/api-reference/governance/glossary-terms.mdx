# Glossary Terms

A **Glossary Term** is a specific business term defined within a glossary. Terms can be hierarchically nested and support synonyms, related terms, and rich metadata.

<Info>
Entity schema follows the [OpenMetadata Standard](https://openmetadatastandards.org/governance/glossary-term/).
</Info>

## Entity Hierarchy

Glossary Terms belong to a Glossary and can be nested:

<CodeGroup>
```txt Glossary Term
Glossary
├── GlossaryTerm (top-level)
│   ├── GlossaryTerm (child)
│   └── GlossaryTerm (child)
└── GlossaryTerm (top-level)
```
</CodeGroup>

## Fully Qualified Name

The FQN format for a glossary term is:

```
glossary.termName
```

For nested terms:

```
glossary.parent.child
```

For example: `BusinessGlossary.Revenue` or `BusinessGlossary.Revenue.GrossRevenue`

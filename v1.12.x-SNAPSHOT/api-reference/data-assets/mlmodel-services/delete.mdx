# Delete an ML Model Service

Delete an ML model service by ID or fully qualified name. Supports soft delete (default), hard delete, and restore operations.

## Delete by ID

<ParamField path="id" type="string" required>
  UUID of the ML model service to delete.
</ParamField>

<ParamField query="recursive" type="boolean" default="false">
  Recursively delete child entities (ML models).
</ParamField>

<ParamField query="hardDelete" type="boolean" default="false">
  Permanently delete the ML model service. If `false`, the service is soft-deleted and can be restored.
</ParamField>

## Delete by Name

Use `DELETE /v1/services/mlmodelServices/name/{fqn}` to delete by fully qualified name.

<ParamField path="fqn" type="string" required>
  Fully qualified name of the ML model service (e.g., `mlflow_svc`).
</ParamField>

<ParamField query="recursive" type="boolean" default="false">
  Recursively delete child entities.
</ParamField>

<ParamField query="hardDelete" type="boolean" default="false">
  Permanently delete the ML model service.
</ParamField>

## Restore a Soft-Deleted ML Model Service

Use `PUT /v1/services/mlmodelServices/restore` to restore a soft-deleted ML model service.

<ParamField body="id" type="string" required>
  UUID of the soft-deleted ML model service to restore.
</ParamField>

## Returns

Soft delete returns the ML model service object with `deleted: true`. Hard delete returns no content (204). Restore returns the restored ML model service object.

---

## Error Handling

| Code | Error Type | Description |
|------|-----------|-------------|
| `401` | `UNAUTHORIZED` | Invalid or missing authentication token |
| `403` | `FORBIDDEN` | User lacks permission to delete this ML model service |
| `404` | `NOT_FOUND` | ML model service with given ID or FQN does not exist |

# Python SDK Reference

The OpenMetadata Python SDK has been redesigned with a modern, fluent API. The previous mixin-based architecture has been replaced by entity-specific classes that provide a consistent interface for all operations.

## Quick Reference

### Configuration

```python
from metadata.sdk import configure

# Explicit credentials
configure(host="http://localhost:8585/api", jwt_token="your-token")

# From environment variables (OPENMETADATA_HOST, OPENMETADATA_JWT_TOKEN)
configure()
```

### Entity Operations

All entity types follow the same pattern:

```python
from metadata.sdk import Tables, Databases, DatabaseSchemas, DatabaseServices
from metadata.sdk import Dashboards, Pipelines, MLModels, Tags, Classifications
from metadata.sdk import Glossaries, GlossaryTerms, Users, Teams

# Create
entity = Tables.create(CreateTableRequest(...))

# Retrieve
entity = Tables.retrieve("uuid")
entity = Tables.retrieve_by_name("service.db.schema.table", fields=["owners", "tags"])

# List with auto-pagination
for table in Tables.list().auto_paging_iterable():
    print(table.name)

# Update
entity.description = "Updated"
Tables.update(str(entity.id), entity)

# Delete
Tables.delete("uuid")
Tables.delete("uuid", recursive=True, hard_delete=True)
```

### Lineage

```python
from metadata.sdk.api import Lineage

# Add lineage
Lineage.add_lineage(
    from_entity_id="source-id",
    from_entity_type="table",
    to_entity_id="target-id",
    to_entity_type="dashboard"
)

# Get lineage
lineage = Lineage.get_lineage("service.db.schema.table", upstream_depth=2, downstream_depth=2)
```

### Error Handling

```python
from metadata.ingestion.ometa.client import APIError

try:
    table = Tables.retrieve("table-id")
except APIError as e:
    if e.status_code == 404:
        print("Not found")
    elif e.status_code == 401:
        print("Authentication failed")
```

## Metadata Ingestion
All connectors are defined as JSON Schemas.
[Here](https://github.com/open-metadata/OpenMetadata/blob/main/openmetadata-spec/src/main/resources/json/schema/entity/services/connections/database/starrocksConnection.json)
you can find the structure to create a connection to StarRocks.
In order to create and run a Metadata Ingestion workflow, we will follow
the steps to create a YAML configuration able to connect to the source,
process the Entities if needed, and reach the OpenMetadata server.
The workflow is modeled around the following
[JSON Schema](https://github.com/open-metadata/OpenMetadata/blob/main/openmetadata-spec/src/main/resources/json/schema/metadataIngestion/workflow.json)

### Configuration Examples
#### Basic Configuration
```yaml
source:
  type: starrocks
  serviceName: starrocks_prod
  serviceConnection:
    config:
      type: StarRocks
      scheme: mysql+pymysql
      username: admin
      password: your_secure_password
      hostPort: starrocks-fe.example.com:9030
```
#### Configuration with Schema Filtering
```yaml
source:
  type: starrocks
  serviceName: starrocks_prod
  serviceConnection:
    config:
      type: StarRocks
      scheme: mysql+pymysql
      username: admin
      password: your_secure_password
      hostPort: starrocks-fe.example.com:9030
      databaseName: analytics
      schemaFilterPattern:
        includes:
          - sales_.*
          - marketing_.*
          - finance_.*
        excludes:
          - .*_test
          - .*_backup
          - temp_.*
      tableFilterPattern:
        includes:
          - fact_.*
          - dim_.*
        excludes:
          - .*_staging
```
#### Configuration with SSL
```yaml
source:
  type: starrocks
  serviceName: starrocks_secure
  serviceConnection:
    config:
      type: StarRocks
      scheme: mysql+pymysql
      username: admin
      password: your_secure_password
      hostPort: starrocks-fe.example.com:9030
      sslConfig:
        caCertificate: /path/to/ca-cert.pem
        sslCertificate: /path/to/client-cert.pem
        sslKey: /path/to/client-key.pem
```
